## 功能迭代记录

### 一、近义词 / 反义词

**数据来源**：Free Dictionary API（`api.dictionaryapi.dev`），已在 `manifest.json` 中授权，无需额外配置。

**实现逻辑**：

1. `background.js` 的 `fetchWordTranslations()` 在解析词典返回结果时，同时收集 `meaning` 级别和 `definition` 级别的 `synonyms` / `antonyms`，去重后各取最多 10 条
2. 翻译写入 Firestore 时新增 `synonyms`、`antonyms` 两个字段（`TRANSLATE_WORD` 消息的 `updateMask` 同步更新）
3. `popup.js` 新增 `renderRelatedWords(wordData)` 函数，在详情弹层翻译区域下方渲染近/反义词标签组

**Firestore 数据结构新增字段**：

```
users/{uid}/words/{wordId}
  + synonyms: array<string>   // 近义词，最多 10 条
  + antonyms: array<string>   // 反义词，最多 10 条
```

**修改文件**：

| 文件            | 改动说明                                                                                                                                                              |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `background.js` | `fetchWordTranslations()` 新增近/反义词收集逻辑；`result` 初始化加 `synonyms[]`、`antonyms[]`；`TRANSLATE_WORD` 消息处理器的 updateMask 和 patchFields 同步新增两字段 |
| `popup.js`      | 新增 `renderRelatedWords(wordData)` 函数；`renderTranslations()` 末尾调用它；`TRANSLATE_WORD` 回调中同步 `wordData.synonyms/antonyms` 到缓存                          |
| `popup.html`    | 详情弹层翻译区域后新增 `<div id="detail-related-words"></div>`                                                                                                        |
| `popup.css`     | 新增 `.related-words-section`、`.related-group`、`.related-label`、`.related-tags`、`.related-tag`、`.related-tag-syn`、`.related-tag-ant` 样式                       |

### 二、CEFR 词汇难度分级（已废弃）

**需求背景**：对收藏的单词标注 A1–C2 难度等级，帮助用户了解词汇水平。

**调研结论**：目前业界**没有免费的公开 CEFR 查询 API**。Cambridge EVP 和 Oxford Learner's Dictionary 均有 CEFR 标注，但均为商业授权，不对外提供接口。

**实现过程**：

1. 初版使用本地 `cefr-data.js`（约 300 词的静态词表）实现展示
2. 列表项在单词名后展示带颜色的等级徽章（A1 绿 → C2 红）
3. 详情页 meta 区域同步展示等级
4. 额外利用 Levenshtein 编辑距离，从词表中查找形近词展示在详情页

**废弃原因**：本地词表覆盖率过低（约 300 词），大多数生词无法匹配到等级，实用性差；且无可靠 API 替代方案。

**最终处理**：

- 删除 `cefr-data.js` 文件
- 从 `popup.js` 移除 `CEFR_DATA` import、`CEFR_COLORS`、`getCefrLevel`、`getCefrBadgeHtml`、`levenshtein`、`getSimilarWords` 所有函数
- 从 `renderRelatedWords()` 移除形近词逻辑和各标签上的 CEFR 角标
- 从 `popup.html` 移除 `<span id="detail-cefr">` 节点
- 从 `popup.css` 移除 `.cefr-badge`、`.cefr-badge-compact`、`.word-text-row`、`.related-tag-sim`、`.related-cefr` 样式

### 三、捐赠方式（支付宝收款码）

**交互设计**：放在设置面板（`⚙️`）底部「支持开发者」分区，不干扰主功能区。

**修改文件**：

## 当前 Firestore 数据结构（完整）

```
users/{uid}/words/{word小写}
  - word: string                    // 单词（小写）
  - displayWord: string             // 原始大小写
  - count: number                   // 收藏次数
  - firstCollectedAt: timestamp
  - lastCollectedAt: timestamp
  - contexts: array[{               // 最近 5 条上下文
      text, url, title, time
    }]
  - contextTranslation: string      // 上下文中文翻译
  - translations: array[{           // 词性释义
      pos, posZh,
      definitions: [{ en, zh }]
    }]
  - synonyms: array<string>         // 近义词（最多 10 条）
  - antonyms: array<string>         // 反义词（最多 10 条）
```
