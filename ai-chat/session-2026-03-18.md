## 项目需求

开发一款 Chrome 插件，功能：

- 在英文网页中选中单词，右键收藏加入生词本
- 同一个词可多次收藏，记录累计收藏次数及每次上下文
- 数据同步到 Google 账号（Firebase Firestore）
- Popup 展示生词本，支持搜索、排序、翻译、导出、清空

## 技术方案

| 模块     | 技术                                             |
| -------- | ------------------------------------------------ |
| 登录认证 | `chrome.identity` + Firebase Auth（Google 登录） |
| 数据存储 | Firebase Firestore（REST API，无需 SDK）         |
| 插件架构 | Manifest V3，Service Worker                      |
| 划词交互 | Content Script 注入                              |

---

## 核心逻辑说明

### 登录流程

1. `chrome.identity.getAuthToken()` 获取 Google access token
2. 用 access token 调用 Firebase REST API 换取 Firebase ID token
3. ID token 存入 `chrome.storage.local`，过期前自动用 refresh token 刷新

### 收藏单词流程

1. Content script 监听文字选中，检测是否为英文单词（正则 `/^[a-zA-Z]([a-zA-Z'-]*[a-zA-Z])?$/`）
2. 右键菜单选择「加入生词本」，发消息给 background：`{ type: 'SAVE_WORD', uid, word, context, pageUrl, pageTitle }`
3. Background 先读取 Firestore 对应文档，若已存在则 `count + 1` 并追加上下文（PATCH with updateMask）；若不存在则全量写入
4. 同时保存最近 5 条上下文（所在句子 + 页面 URL）

### Firestore 数据结构

```
users/{uid}/words/{word小写}
  - word: string（小写）
  - displayWord: string（原始大小写）
  - count: number（收藏次数）
  - firstCollectedAt: timestamp
  - lastCollectedAt: timestamp
  - contexts: array[{ text, url, title, time }]（最近5条）
  - contextTranslation: string（上下文中文翻译）
  - translations: array[{ pos, posZh, definitions: [{ en, zh }] }]
```

### Firestore 安全规则

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/words/{wordId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

## manifest.json 关键配置

```json
{
  "manifest_version": 3,
  "oauth2": {
    "client_id": "xxx.apps.googleusercontent.com",
    "scopes": [
      "https://www.googleapis.com/auth/userinfo.email",
      "https://www.googleapis.com/auth/userinfo.profile",
      "https://www.googleapis.com/auth/datastore"
    ]
  },
  "permissions": ["storage", "identity", "contextMenus", "activeTab", "scripting"],
  "host_permissions": ["https://api.dictionaryapi.dev/*"],
  "background": { "service_worker": "background.js", "type": "module" }
}
```

> ⚠️ `"type": "module"` 使 background.js 可以使用 `import`

## 已知注意事项

1. **首次登录**会出现「谷歌尚未验证此应用」警告 → 点「先进的」→「继续」即可，开发阶段正常现象
2. **Firebase 授权域**不需要添加 `chrome-extension://` — 插件用 `chrome.identity` 走的是独立通道
3. **OAuth Client ID 类型**必须是「Chrome 应用」，不能用 Firebase 自动生成的「Web 客户端」

## 本地开发更新方式

1. 修改代码
2. 打开 `chrome://extensions`
3. 点「新词本」卡片上的 🔄 刷新按钮

> ⚠️ Chrome 直接读取本地文件夹，不要删除或移动文件夹，否则插件失效

如需移动文件夹：

```bash
mv ~/Desktop/new-word ~/Documents/new-word
# 然后在 chrome://extensions 删除旧插件，重新「加载已解压的扩展程序」
```

## 2026-03-18 功能更新：接入 Google 翻译

### 需求

收藏单词时自动翻译成中文，结合上下文，显示多个词义，标注词性。

### 新增技术依赖

| 服务                                          | 用途                     | 费用                                         |
| --------------------------------------------- | ------------------------ | -------------------------------------------- |
| Free Dictionary API (`api.dictionaryapi.dev`) | 获取英文词条、词性、释义 | 免费，无需 key                               |
| Google Cloud Translation API v2 (Basic)       | 批量翻译为中文           | 有免费额度（50万字符/月），超出 $20/百万字符 |

> ⚠️ 使用翻译功能前，需到 Google Cloud Console 启用 **Cloud Translation API**：
> https://console.cloud.google.com/apis/library/translate.googleapis.com
> 启用后直接使用现有 `FIREBASE_CONFIG.apiKey`，无须额外密钥。

### 修改文件

- `manifest.json`：`host_permissions` 新增 `https://api.dictionaryapi.dev/*`
- `firebase-config.js` / `firebase-config.example.js`：增加启用 Translation API 的提示注释
- `background.js`：
  - 新增 `POS_ZH_MAP`（词性中英映射）
  - 新增 `fetchWordTranslations(word, context)` 函数
  - `saveWord()` 首次收藏时自动获取翻译写入 Firestore；已有词时只更新上下文翻译
  - 新增 `TRANSLATE_WORD` 消息处理器（供 popup 对老数据手动触发翻译）
- `popup.html`：详情弹层加入 `#detail-translations` 区域
- `popup.js`：
  - `renderWordList` 列表项显示 "名词 · 意思1；意思2" 预览行
  - 新增 `getTranslationPreview(wordData)` 函数
  - `openDetail` 调用 `renderTranslations(wordData)`
  - 新增 `renderTranslations(wordData)` 函数（无翻译时显示"获取中文翻译"按钮）
- `popup.css`：新增翻译 UI 样式（`.pos-badge`、`.def-zh`、`.def-en`、`.translation-context` 等）

### Firestore 数据结构变更（新增字段）

```
users/{uid}/words/{wordId}
  ...（原有字段）
  + contextTranslation: string        // 上下文句子的中文翻译
  + translations: array[{
      pos: string,                    // 词性（英文，如 "noun"）
      posZh: string,                  // 词性（中文，如 "名词"）
      definitions: array[{
        en: string,                   // 英文释义（来自词典）
        zh: string                    // 中文翻译
      }]
    }]
```

### 交互逻辑

1. 划词收藏 → `background.js` 先保存单词，同时调用 Dictionary API + Google Translate → 一并写入 Firestore
2. 老数据（无 `translations` 字段）→ 进入详情页显示「获取中文翻译」按钮 → 点击后后台翻译 → 写回 Firestore 并刷新 UI
3. 生词列表展示：单词下方有一行蓝紫色小字"名词 · 意思1；意思2"
4. 详情弹层：按词性分组展示中文释义 + 英文原文；下方附上下文翻译（斜体引号样式）
